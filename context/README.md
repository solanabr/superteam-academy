# Context Index

Use this folder as the working memory for planning and execution.

## Files

- `PLAN.md` — master roadmap and execution order
- `STATUS.md` — current project snapshot (done / in progress / next)
- `BACKEND-PRODUCTION.md` — backend productionization checklist
- `FRONTEND-PHASES.md` — frontend implementation guidelines, phases, and route map
- `DESIGN-GUIDELINES.md` — UI/UX and visual system guidelines for frontend execution
- `BOUNTY-MAPPING.md` — requirement-to-phase mapping for bounty scope and deliverables
- `DECISIONS.md` — architectural and product decisions that guide implementation
- `PR-CHECKLIST.md` — pre-push and pre-PR quality gate
- `API-IO.md` — request/response IO reference for endpoint testing
- `postman/Superteam-Academy.postman_collection.json` — importable Postman collection for all endpoints
- `postman/Superteam-Academy.local.postman_environment.json` — local environment variables for Postman

## Update Rule

When a phase/task changes, update:

1. `STATUS.md`
2. `PLAN.md`
3. The relevant detailed file (`BACKEND-PRODUCTION.md`, `FRONTEND-PHASES.md`, `DESIGN-GUIDELINES.md`, or `BOUNTY-MAPPING.md`)
