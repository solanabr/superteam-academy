# CMS Guide

Sanity CMS integration for managing course content. The integration is optional — the app falls back to hardcoded data in `src/data/` when Sanity is not configured.

## Setup

### 1. Create a Sanity Project

Go to [sanity.io/manage](https://www.sanity.io/manage) and create a new project. Note the project ID.

### 2. Configure Environment Variables

```env
NEXT_PUBLIC_SANITY_PROJECT_ID=your-project-id
NEXT_PUBLIC_SANITY_DATASET=production
SANITY_API_TOKEN=              # Required for Studio mutations
SANITY_REVALIDATE_SECRET=      # Optional: webhook-based revalidation
```

### 3. Access Sanity Studio

The app embeds Sanity Studio at `/studio`. Navigate to `http://localhost:3000/studio` to manage content.

## Content Schema

Schemas are defined in `sanity/schemas/` and registered in `sanity/schemas/index.ts`.

### Course (document)

The top-level content type. Each course contains modules, which contain lessons.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | Yes | Course title |
| `slug` | slug | Yes | URL slug (auto-generated from title) |
| `description` | text | No | Short description (catalog card) |
| `longDescription` | text | No | Full description (course detail page) |
| `difficulty` | string | Yes | `Beginner`, `Intermediate`, or `Advanced` |
| `topic` | string | No | Category: Core, Framework, DeFi, Security |
| `topicLabel` | string | No | Uppercase badge label |
| `duration` | string | No | Total duration (e.g., `4h`) |
| `xp` | number | No | Total XP awarded for completing all lessons |
| `accent` | string | No | Hex color for UI theming (e.g., `#34d399`) |
| `icon` | string | No | Lucide icon name (see below) |
| `codePreview` | string[] | No | Lines of code shown on course card |
| `instructor` | reference | No | Reference to Instructor document |
| `order` | number | No | Sort position (lower = first) |
| `modules` | module[] | No | Ordered list of modules |
| `reviews` | object[] | No | Student reviews (name, rating 1-5, text) |

Available icon values: `Blocks`, `Anchor`, `Landmark`, `Shield`, `Wallet`, `Coins`, `TestTube`, `ArrowLeftRight`, `GitBranch`

### Module (object)

Nested within a Course. Groups related lessons.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | Yes | Module title |
| `lessons` | lesson[] | No | Ordered list of lessons |

Each module gets a `_key` auto-generated by Sanity, used as the module ID in the frontend.

### Lesson (object)

Nested within a Module. Three types supported.

| Field | Type | Required | Shown When | Description |
|-------|------|----------|-----------|-------------|
| `title` | string | Yes | Always | Lesson title |
| `duration` | string | No | Always | Duration (e.g., `8 min`) |
| `type` | string | Yes | Always | `video`, `reading`, or `challenge` |
| `markdown` | text | No | Always | Full lesson content in markdown |
| `starterCode` | text | No | type = challenge | Pre-filled code in Monaco editor |
| `solutionCode` | text | No | type = challenge | Revealed solution code |
| `hints` | string[] | No | type = challenge | Progressive hints |
| `testCases` | testCase[] | No | type = challenge | Automated test cases |

Challenge-specific fields (`starterCode`, `solutionCode`, `hints`, `testCases`) are hidden in Studio when the lesson type is not `challenge`.

### Test Case (object)

Nested within a Lesson of type `challenge`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Test name displayed in UI |
| `input` | text | No | Input passed to the test |
| `expected` | text | No | Expected output for comparison |

### Instructor (document)

Standalone document, referenced by courses.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Full name |
| `role` | string | No | Title (e.g., "Senior Developer") |
| `bio` | text | No | Short biography |
| `avatar` | image | No | Profile photo (supports hotspot cropping) |

## GROQ Queries

All queries are defined in `lib/sanity.ts` and used by fetchers in `lib/sanity-fetch.ts`.

### `allCourses` — Catalog Listing

Returns all courses ordered by `order` field. Includes computed `moduleCount` and `lessonCount`.

```groq
*[_type == "course"] | order(order asc) {
  title,
  "slug": slug.current,
  description,
  difficulty,
  duration,
  xp,
  accent,
  "instructor": instructor->{name, role},
  "moduleCount": count(modules),
  "lessonCount": count(modules[].lessons[])
}
```

### `courseBySlug` — Course Detail

Returns a single course with full module/lesson structure and reviews.

```groq
*[_type == "course" && slug.current == $slug][0] {
  title,
  "slug": slug.current,
  description,
  longDescription,
  difficulty,
  topic,
  topicLabel,
  duration,
  xp,
  accent,
  icon,
  codePreview,
  "instructor": instructor->{name, role, bio, avatar},
  modules[] {
    _key,
    title,
    lessons[] {
      _key,
      title,
      duration,
      type
    }
  },
  reviews[] {
    name,
    rating,
    text
  }
}
```

### `lessonContent` — Lesson Content

Returns lesson content for a specific lesson within a course, filtered by `_key`.

```groq
*[_type == "course" && slug.current == $courseSlug][0] {
  modules[] {
    lessons[_key == $lessonId] {
      _key,
      title,
      type,
      markdown,
      starterCode,
      solutionCode,
      hints,
      testCases[] {
        name,
        input,
        expected
      }
    }
  }
}
```

## Using the Fetchers

```typescript
import { getAllCourses, getCourse, getLesson } from "@/lib/sanity-fetch";

// Catalog page — returns CourseDetail[]
const courses = await getAllCourses();

// Course detail page — returns CourseDetail | null
const course = await getCourse("solana-fundamentals");

// Lesson page — returns LessonContent | null
const lesson = await getLesson("solana-fundamentals", "lesson-key-abc", "en");
```

The `fetchFromCMS()` wrapper in `lib/sanity.ts` handles:
- Returning `null` if `sanityClient` is not initialized (no project ID)
- Catching errors silently (returns `null`)
- CDN caching (`useCdn: true`)

## Creating a Course (Step by Step)

### In Sanity Studio (`/studio`)

1. **Create an Instructor** (if new)
   - Go to Instructor → Create
   - Fill in name, role, bio, avatar
   - Publish

2. **Create the Course**
   - Go to Course → Create
   - Fill in required fields: title, slug, difficulty
   - Set the accent color (hex), topic, duration, XP
   - Choose an icon name from the supported list
   - Link the instructor reference
   - Set the sort order

3. **Add Modules**
   - In the course editor, add items to the Modules array
   - Each module needs a title

4. **Add Lessons to Each Module**
   - Within each module, add lesson items
   - Set the type: `video`, `reading`, or `challenge`
   - For readings: add markdown content
   - For challenges: add starterCode, solutionCode, hints, and test cases

5. **Add Reviews** (optional)
   - Add review objects with reviewer name, rating (1-5), and text

6. **Publish**
   - Click Publish — changes appear immediately (CDN cached)

### In Hardcoded Data (Development)

Edit `src/data/courses.ts` and add a new entry to the `courses` array:

```typescript
{
  slug: "my-course",
  title: "My Course",
  description: "Short description",
  longDescription: "Full description",
  difficulty: "Beginner",
  topic: "Core",
  topicLabel: "CORE",
  duration: "3h",
  lessons: 12,
  completed: 0,
  xp: 600,
  accent: "#10b981",
  icon: Blocks,
  codePreview: ["fn main() {", '    println!("Hello");', "}"],
  instructor: { name: "Jane Doe", role: "Developer" },
  modules: [
    {
      id: "m1",
      title: "Getting Started",
      lessons: [
        { id: "l1", title: "Introduction", duration: "5 min", type: "reading", completed: false },
        { id: "l2", title: "First Challenge", duration: "15 min", type: "challenge", completed: false },
      ],
    },
  ],
  reviews: [],
}
```

Add lesson content in `src/data/lesson-content.ts` (and locale variants).

## Fallback Behavior

When `NEXT_PUBLIC_SANITY_PROJECT_ID` is not set:

1. `sanityClient` is `null` (no Sanity client created)
2. `fetchFromCMS()` returns `null` immediately
3. `getAllCourses()` returns `[]` (empty array)
4. `getCourse()` returns `null`
5. Pages use `getCourseBySlug()` from `data/courses.ts` as the primary source
6. Course detail pages: hardcoded data renders immediately, Sanity fetch is a no-op

This means the app is fully functional without any CMS setup.

## Translation Keys

Course content uses translation keys with the pattern:

```
courseContent.{slug}.title           → Course title
courseContent.{slug}.longDescription → Full description
courseContent.{slug}.{moduleId}      → Module title
courseContent.{slug}.{lessonId}      → Lesson title
courseContent.{slug}.r{index}        → Review text (r0, r1, r2...)
```

These keys must exist in all locale files (`en.json`, `pt-BR.json`, `es.json`) for translated content. If a key is missing, the key path itself is displayed as fallback.
