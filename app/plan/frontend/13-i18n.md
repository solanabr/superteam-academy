# i18n Service

## Overview

The i18n Service provides multi-language support for the platform. All UI strings are externalized and support PT-BR, ES, and EN from day one.

## Supported Languages

| Code | Language | Native |
|------|----------|--------|
| `en` | English | English |
| `pt-BR` | Portuguese (Brazil) | Português |
| `es` | Spanish | Español |

## Implementation

### 1. next-intl Configuration

```typescript
// i18n/config.ts
export const locales = ['en', 'pt-BR', 'es'] as const;
export type Locale = typeof locales[number];

export const defaultLocale: Locale = 'en';

export const localeNames: Record<Locale, string> = {
  'en': 'English',
  'pt-BR': 'Português',
  'es': 'Español',
};
```

```typescript
// i18n/request.ts
import { getRequestConfig } from 'next-intl/server';
import { locales, Locale } from './config';

export default getRequestConfig(async ({ locale }) => {
  if (!locales.includes(locale as Locale)) {
    locale = 'en';
  }
  
  return {
    messages: (await import(`./messages/${locale}.json`)).default,
  };
});
```

### 3. Language Switcher Component

```typescript
// components/i18n/LanguageSwitcher.tsx
'use client';

import { useLocale } from 'next-intl';
import { usePathname, useRouter } from 'next/navigation';
import { locales, localeNames, Locale } from '@/i18n/config';
import { Button } from '@/components/ui/button';
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from '@/components/ui/dropdown-menu';

export function LanguageSwitcher() {
  const locale = useLocale() as Locale;
  const pathname = usePathname();
  const router = useRouter();
  
  const changeLocale = (newLocale: Locale) => {
    const segments = pathname.split('/');
    segments[1] = newLocale;
    router.push(segments.join('/'));
  };
  
  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button variant="ghost" size="sm">
          <GlobeIcon className="mr-2 h-4 w-4" />
          {localeNames[locale]}
        </Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent>
        {locales.map((loc) => (
          <DropdownMenuItem
            key={loc}
            onClick={() => changeLocale(loc)}
            className={loc === locale ? 'bg-accent' : ''}
          >
            {localeNames[loc]}
          </DropdownMenuItem>
        ))}
      </DropdownMenuContent>
    </DropdownMenu>
  );
}
```

### 4. Middleware for Locale Detection

```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';
import { locales, defaultLocale } from './i18n/config';

export default createMiddleware({
  locales,
  defaultLocale,
  localePrefix: 'always',
});

export const config = {
  matcher: ['/((?!api|_next|_vercel|.*\\..*).*)'],
};
```

### 5. Usage in Components

```typescript
// Example usage
import { useTranslations } from 'next-intl';

export function CourseCard({ course }) {
  const t = useTranslations('courses');
  
  return (
    <div className="course-card">
      <h3>{course.title}</h3>
      <p>{t('lessons', { count: course.lessonCount })}</p>
      <p>{t('xp', { xp: course.xpTotal })}</p>
      <span className="difficulty">
        {t(`difficulty.${course.difficulty}`)}
      </span>
      <Button>{t('enroll')}</Button>
    </div>
  );
}
```

## Course Content Note

Course content can remain in its original language. The i18n system handles:
- UI strings
- Navigation
- Error messages
- Notifications

The CMS can store content in multiple languages, and the frontend can display based on user preference.
